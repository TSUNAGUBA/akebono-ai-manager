## 主な指摘と対応(抜粋)

### Critical(ラウンド1)

1. **対話ターン保存のサイレント全損**: `WHERE created_at = $2` が node-postgres の timestamptz 丸め(µs→ms)で恒常不成立
   → WHERE を dialogue_id 単独に変更+0行更新は AIM-2002 で顕在化
2. **embedding の一括リクエスト**: gemini-embedding-001 は 1リクエスト1インスタンスのみ対応(公式ドキュメントで裏取り)
   → 逐次呼び出しに変更
3. **dashboard_ro の権限とページクエリの矛盾**: 概要ページが生対話ログを直接参照し本番で常時 500
   → 発話内容を含まない集計専用ビュー `ops.v_dialogue_daily_stats` を新設(プライバシー境界は維持)

### Major(ラウンド1)

- Secret Manager 登録値への改行混入(PowerShell パイプ仕様)→ 一時ファイル経由に修正
- ドキュメントの回復手順と実装の乖離 → repeatable マイグレーションを「毎回適用・冪等」に変更し、
  ロール後作成・pg_cron 後有効化とも db-migrate 再実行だけで反映される設計に
- 朝の問いかけ配信失敗時の補償なし(再実行しても永遠に未配信)→ 補償削除を追加
- dashboard の生 URL 公開(header モード時のなりすまし余地)→ ingress を LB(IAP)経由のみに制限
- batch の呼び出し元検証をフェイルクローズ化(BATCH_INVOKER_SA 必須)

### 検証方法の特記

- レビュアーは pg の timestamptz 丸めをローカルで実測、embedding 制約を公式ドキュメントで裏取りした上で指摘
- 各ラウンドで `npm run build` / `npm test`(56件)の通過を確認
- ダッシュボードはモック DB での全ページレンダリング+モバイル(390px)エミュレーションで検証

## 残存事項(指摘外の申し送り)

- **監査官の前提確認(ラウンド2 Minor)**: versioned マイグレーション(0001/0003)を適用済みの DB が存在しないこと。
  → **確認済み**: マイグレーションは本ブランチで新規作成され、どの DB にも未適用(デプロイパイプラインは未稼働、
  本開発環境に DB 接続情報は存在しない)。以後は「適用済み versioned の変更禁止」を徹底する
- `callVertex` に明示タイムアウトなし → Phase 1 の対話レイテンシ実測(要件 6.3)とあわせて監視
- 振り返りの帰属は「直近の朝の対話」基準(保守側)。運用で outcome 捕捉率が低い場合は
  `AND m2.hypothesis IS NOT NULL` の1行変更で「直近の朝仮説」基準に調整可能
