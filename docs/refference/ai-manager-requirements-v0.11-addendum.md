# AI マネージャー要件定義 v0.11 追補 — ナレッジ同期の診断性・PDF 対応・ファイル名規約撤廃・管理 UI フォーム改善

> **位置づけ:** 本書は `ai-manager-requirements-design.md`(v0.2)への**追補**であり、v0.2 本文は変更しない。
> 既存の追補(v0.3〜v0.10)と本書が矛盾する場合は本書(v0.11)を優先する。
> **背景:** オペレーター報告・要望(2026-07-13)
> ①「ナレッジの Google ドライブとの同期が行われない」(secret のフォルダ ID の場所と、同期したいフォルダの2画面を提示)
> ②「ファイルアップロードのファイル名規約は何のためにあるのか。不要なら除外してほしい」
> ③「ナレッジのファイル形式は PDF にも対応してほしい」
> ④「プロジェクト編集フォームの textarea の配置・マイルストーン入力の縦位置が崩れている。登録後にページ最上部へ戻される。他ページも含め一括で修正してほしい」

## 1. 背景と課題

| # | 課題 | 現状 |
|---|---|---|
| C14 | 同期対象が「システムから何が見えているか」を確認する手段がない | 同期が 0 件でもエラーにならず、原因(フォルダ ID のずれ・SA への共有漏れ・ショートカット)を切り分けられない |
| C15 | Drive の**ショートカット**を辿れない | 既存フォルダをショートカットでナレッジフォルダに置く構成では、配下が列挙されず静かに同期 0 件になる |
| C16 | ファイル名規約(半角小文字英数字のみ)が投入の障害 | 規約は歴史的なもの(URL クエリでの受け渡し・Drive クエリの単純化)で、同期自体はファイル名の文字種に依存しない。日本語名の実ファイルを投入できない |
| C17 | PDF のナレッジを扱えない | `.md` / `.txt` のみ。取引基準等の PDF 文書を検索対象にできない |
| C18 | 管理 UI フォームの操作感 | textarea が form-grid 内で他の入力と横並びになり崩れる。date 入力が未スタイルで縦位置がずれる。PRG リダイレクトで毎回ページ最上部へ戻される |

## 2. ナレッジ同期の診断性とショートカット対応(C14 / C15)

- **ショートカットの解決**: 一覧・同期はショートカット(`application/vnd.google-apps.shortcut`)を
  実体へ解決して辿る。フォルダショートカットは配下を再帰列挙し、ファイルショートカットは実体 ID で
  取り込む(doc_id は実体 ID — 実体・ショートカットの両方が見える場合は重複させず実体を優先)
- **共有の制約の可視化**: ショートカットは Drive の共有権限を**引き継がない**。実体側が SA に
  共有されていない場合は全体を止めず(原則4)、①ナレッジ管理ページに警告バナー
  ②同期ログに警告、として可視化する
- **チャンクの保護(原則2)**: アクセスできないショートカットがある間は、削除済み文書のチャンク掃除を
  スキップする(見えない配下を「削除された」と誤判定して既存チャンクを消さない)
- **設定ずれの自己診断**: ナレッジ管理ページに「同期対象フォルダ: Drive で開く」リンク
  (`KNOWLEDGE_DRIVE_FOLDER_ID` の場所)を表示する。一覧に表示されないファイルは同期されない、
  という関係をページ上に明示する
- **同期 0 件の警告**: 同期実行時にファイルが 0 件の場合、フォルダ ID・共有の確認手順つきの
  警告ログを出す(静かに何もしない状態を作らない)
- ナレッジからの削除(ゴミ箱移動)は、**ファイルショートカット**ではショートカット側を
  ゴミ箱へ移動する(実体は元の場所に残す — 意図は「ナレッジからの除外」であり元データの削除ではない)。
  **フォルダショートカット先の中にある実ファイル**の削除は、従来どおり実体のゴミ箱移動になる
  (共有元フォルダからもゴミ箱へ移る。復元は可能)— 元データを残したい場合は削除ではなく
  実体側での移動・共有解除で対応する
- 警告バナーの対象は**フォルダ**ショートカットの到達失敗。実体が未共有の**ファイル**
  ショートカットは一覧には出るが、同期時にファイル単位の失敗(failed 件数+エラーログ)になる

## 3. ファイル名規約の撤廃(C16)

- ファイル名本体の文字種制限(半角小文字英数字・ドット・ハイフン・アンダースコア)を**撤廃**し、
  日本語等の Unicode 名をそのまま許容する
- 禁止は表示・API 呼び出しを壊し得るもののみ: 制御文字・パス区切り(`/` `\`)。
  長さは拡張子込み 128 文字以内(従来どおり)
- 英大文字は小文字へ変換して保存する(従来投入分との同名上書き互換を保つ)
- 失敗ファイル名のフラッシュ表示はカンマ区切りから **JSON 配列**での受け渡しに変更する
  (日本語名にカンマ等が含まれ得るため。表示時は HTML エスケープ+規約検証)
- **旧規約の目的(記録)**: ①失敗ファイル名を URL クエリで受け渡す際の表示注入防止の単純化
  ②Drive 検索クエリのエスケープの単純化 ③フォルダ規約(customer/{顧客ID} 等)との統一感。
  いずれも同期の動作要件ではなく、エスケープ・検証で代替可能なため撤廃する

## 4. PDF 対応(C17)

- **投入(ファイルアップロード)**: `.pdf` を受け付け、**バイナリのまま** Drive へ保存する
  (SoT は Drive・投入経路は増やすだけ、という v0.4/v0.6 の原則を維持)。
  1ファイル 3MB 以内(multipart 全体の 4MB 上限内)。マジックバイト(`%PDF-`)を投入前に検証する
- **同期(テキスト抽出)**: knowledge-sync が PDF のテキスト層をバッチ内でローカル抽出し、
  既存のチャンク分割 → content_hash 差分 → embedding フローにそのまま乗せる(ADR-17)
- **テキスト層のない PDF**(スキャン画像のみ)は抽出結果が空になるため同期をスキップし、
  警告ログで知らせる(OCR は行わない — スコープ外)
- 直接入力フォームは従来どおりテキスト(.md / .txt)専用(.pdf 名の指定はエラー)
- 同期時の抽出サイズ上限は 20MB(Drive 直置きの大きな PDF によるメモリ枯渇の防止。超過はスキップ+警告)

## 5. 管理 UI フォームの一括改善(C18)

全マスタ管理ページ(業界・顧客・顧客間関係・プロジェクト・ユーザー・ナレッジ・状況確認)を対象に:

- **textarea の独立行化**: form-grid 内の textarea(プロジェクトの目的・内容、顧客のメモ)は
  他の入力と横並びにせず**行幅いっぱいの独立行**にする。高さは5行分を確保し、
  リサイズ不可・スクロール可とする(ナレッジの本文入力のみ長文前提のため 10 行)
- **入力の縦位置の統一**: `input[type="date"]` / `input[type="file"]` にもテキスト入力と同じ
  ブロックスタイルを適用する(マイルストーンの「マイルストーン名」と「期日(任意)」の
  高さ・上下位置のずれを解消)
- **PRG リダイレクトのアンカー**: 登録・更新後にページ最上部へ戻さない。操作したセクションへ
  `#fragment` で戻る(プロジェクトのマイルストーン操作 → #milestones、タスク進捗 → #tasks、
  各マスタの追加 → 追加フォーム、ナレッジの投入 → 投入フォーム)。
  編集の完了(update)は従来どおり一覧先頭へ戻る(編集モードの終了として自然なため)
- **結果バナーの追従**: フラッシュバナー(保存結果・エラー)は sticky 化し、アンカーで
  ページ中程に遷移しても結果が視界に入るようにする(モバイルはヘッダー高が可変のため追従しない)

## 6. スコープ外(本追補では実施しない)

- スキャン PDF の OCR、Office 文書(Word / Excel)の投入
- 顧客マスタの「ナレッジ Drive フォルダID」欄の同期への利用(参考情報のまま。
  同期対象は共通ナレッジフォルダの customer/{顧客ID}/ 規約 — 欄の説明文で明示する)
- ナレッジ一覧のページング・検索(件数が問題になった時点で検討)

## 7. 受け入れ基準

1. ナレッジ管理ページから「システムが実際に読んでいるフォルダ」を Drive で開いて確認できる
2. ショートカット先にアクセスできない場合、警告がページに表示され、既存チャンクは削除されない
3. 日本語名のファイル(例: `PB商品生産工場の認定.md`)をそのまま投入・同期できる
4. テキスト層のある PDF を投入すると、同期後に Chat の検索(RAG)で内容が参照される
5. テキスト層のない PDF は同期時にスキップされ、警告ログに残る(エラーで同期は止まらない)
6. プロジェクト編集フォームの textarea は独立行・5行・リサイズ不可、マイルストーンの入力欄は
   縦位置が揃い、マイルストーン登録後もページ最上部へ戻されない
7. 下位互換: 既存の投入済みファイル(小文字英数字名)の上書き・削除・同期は従来どおり動作する
   (スキーマ変更なし。データ更新パッチ不要)
